03 — UEN Lookup Deep Dive

Why we replaced AI with
direct source lookup

The previous system used Gemini AI to search the web and guess the UEN. This costs tokens per lead and introduces hallucination risk — the AI is inferring rather than reading authoritative data.


Our approach: query sgpbusiness.com directly — the official ACRA-linked Singapore Business Directory — using Jina AI to read the page, then score results by name match. Free, accurate, and verifiable.

09 — Research Findings

What we learned building this

This system was built independently from the existing one — for research, stress testing, and cost optimization. Here is what the research surfaced.

🔢
300 leads is the tested baseline per query
Current tested limit via Apify API call is 300 results per query. Apify integrated natively in n8n shows promise for higher limits (potentially 1,000+) but requires budget approval to test since each run has a real cost. The 300 figure is a confirmed baseline, not a hard ceiling.
🎯
Query diversity = more unique leads
Running "cleaning services Singapore" twice returns largely the same 300. Adding a city variable — "cleaning services Jurong" vs "cleaning services Tampines" — produces significantly less overlap. The form now accepts a City field specifically for this reason.
🔍
Deduplication must happen after scraping
Google Maps cannot be pre-filtered with "don't show me businesses I already have." The query runs first, results come back, then we deduplicate. This means Apify cost is spent before knowing how many are new — unavoidable by design.
🤖
AI adds cost without accuracy gain for UEN
Gemini web search for UEN costs tokens per lead and sometimes hallucinates. sgpbusiness.com is the canonical ACRA data source — querying it directly via Jina is free, faster, and more reliable.
📱
Landline vs mobile filtering is critical
Without a mobile-only filter, SG landlines (starting with 3 or 6) get mixed with WhatsApp-eligible mobiles (starting with 8 or 9). The previous system had a bug where this filter was broken due to the +65 prefix not being stripped correctly.
📞
Google Maps phone is surface-level only
Google Maps lists whatever phone number a business registered — often a landline. The system separates this into two columns: Map Phone (raw from Google) and Website number (validated SG mobile scraped from the business website). If both are empty, the lead has no publicly available mobile and requires manual outreach or the future DM finder workflow.
⚡
Isolated workflows = resilient to n8n runtime limits
n8n cloud has execution time limits. A single workflow processing 300 leads end-to-end risks hitting these limits and losing progress. Separate scheduled workflows process one lead at a time and retry automatically.

11 — Known Limitations

What this system cannot do (yet)

⚠️ 300 Lead Baseline Per Query
Current tested limit via Apify API call is 300 results per search query. Not yet confirmed as a hard ceiling -- Apify natively integrated in n8n may support higher limits up to 1,000+.
→ Short term: Use city + industry keyword combinations to maximize unique leads per run. Long term: Pending budget approval to test 1,000 lead runs via native n8n Apify integration.
⚠️ Dedup Cost is Unavoidable
Google cannot be pre-filtered to exclude known businesses. Apify credit is spent before deduplication, meaning some credits go to leads already in the sheet.
→ Workaround: Vary queries to minimize overlap. Future: pre-filter using place_id matching
⚠️ SerpAPI Free Tier Limit
100 free SerpAPI searches per month. Currently exhausted — refreshes at the start of next billing cycle. Carousell checks are paused until then.
→ Workaround: Upgrade to $50/mo Hobby plan for 5,000 searches, or treat Carousell check as optional per AM
⚠️ Website Timeouts
Some SG business websites are slow or block scrapers. Jina enforces a 10-second timeout — leads with inaccessible websites may have empty email/phone fields.
→ Workaround: Continue-on-fail is enabled — timeout leads are skipped gracefully and can be manually enriched
⚠️ UEN Match Confidence
Very generic company names (e.g. "Renovation House") may have low scoring confidence. System writes null rather than guess wrong — these rows show empty UEN.
→ Workaround: Minimum score threshold prevents false UENs. Low-confidence rows can be manually verified
⚠️ SG Only (Currently)
UEN lookup uses sgpbusiness.com which is SG-specific. HK leads would need a different ACRA-equivalent source for company registration numbers.
→ Workaround: HK expansion is on the roadmap. Apify and contact enrichment already support HK queries
